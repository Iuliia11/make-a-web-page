Lesson 1: The basics of the web

The world wide web is a collection of HTML (Hyper Text Markup Language) files. HTML is a main type of a content which can be a plain text, images, music, videos etc. Each HTML document is linked to another one, and they all together compose a web like it is.

Major pieces

How it works: a user by means of computer runs a browser (EE/Chrome/Safari etc.), which makes a request to servers through the Internet. Servers are computers that host files that make up the web. The Internet turns to servers using http protocols (the main protocol of the web) and servers respond with files which user's browser displays.

HTML

HTML documents are the heart of the web. HTML is made up of:

  • text content (what you see);
  • markup (what it looks like);
  • references to other documents (images, videos etc.);
  • links to other pages.
Tags

HTML markup is made up of tags: 'Opening tag' (in angle brackets) Content '/Closing tag' (in angle brackets). Most of HTML tags come in pairs, but some of them are unpaired and called void (they don't have a content, so they don't need a closing tag, for example,'img' and 'br').

Elements

An HTML element refers to everything within a set of opening and closing tags.

Attributes

HTML elements can have attributes. Attributes always follow an openning tag. They look like: attribute name="value".

Inline/Block Tags

There are inline and block tags. Inline tags just end a line, and block tags make an ivisible box around the content. Block tags have height and width. The other one example of inline/block tags: 'span' is inline and 'div' is block.

Exploring the web

Tree-like structure

A branching structure which a browser builds called a "tree-like structure". It comes from the fact that HTML elements can have other elements inside them. This can be a text, or it can be another element. Each element has an opening tag and a closing tag.
DOM (Document Oriented Model) is a cross-plarform and language-independent convention for representing and interacting with objects in HTML.
The nodes of every document are organized in a tree structure, called the DOM tree. The browser turns HML tags into elements that form a tree.

Text Editors

A text editor is a program that helps you type text similar to document editors, such as Microsoft Word or Corel WordPerfect, but without all the extra formatting functions that document editors have. The most popular text editors are Codepen, Stratchpad and Sublime. Strachpad has immediate feedback and great for beginner, but has only HTML and CSS. Codepen is great for code sharing (you can just "copy-paste" code from Stratchpad to Codepen and share it), but requires an account. It has the following languages: HTML, CSS, Javascript. Sublime is difficult for beginners, but has a high proffesionalism and a variety of languages.

Lesson 3: Structured HTML with CSS styling.

Do not repeat yourself

Avoiding repetition is very important for a programmer. Do not repeat yourself (DRY) is a principle of software development, aimed at reducing repetition of information of all kinds. When the DRY principle is applied successfully, a modification of any single element of a system does not require a change in other logically unrelated elements. If you do not keep on this principle, then every time you make some changes everywhere the repeated code appears, errors will arise.

CSS

CSS stands for Cascading Styles Sheets. CSS is a style sheet language used for descibing the look and formatting of a document written in a markup language. CSS allows you to use a specific syntax and rules how elemens look on the page, the size of the font, the color, background, borders, position etc.
"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.
You can also think about CSS as a search and replace tool: you identify a class or a tag of the element you want to find (or match, in CSS terminology), and then what you want to do with it, or what property values to replace with different ones.

Order Matters

It also matters where you define the rules and in what order they are applied. Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions:

  1. the default style of a browser (different browsers have slightly different styles);
  2. stylesheet in a separate file (this is what you will be mostly using);
  3. stylesheet inside HTML (this can be done for small projects but is not ideal);
  4. inline style in an element (this can also be done but should be avoided).
Inheritance

This is an important idea that is worth thinking about. Inheritance is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants.Descendant elements may inherit CSS property values from any ancestor element enclosing them. In general, descendant elements inherit text-related properties, but box-related properties are not inherited.
Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing the software developers to write less CSS.

Boxes

There are four main points about box sizing:

  • HTML elements are boxes and each box has 4 components (content, padding, border and margin).
  • Because there are so many components to each box, it can often be hard to get the size of a box just right.
  • There are two techniques you can use to help deal with sizing issues.
    1. Set sizes in terms of percentages rather than pixels
    2. Set the box-sizing attribute to border-box for every element.
  • Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.
Box Positioning
  • Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  • Adding the rule 'display: flex;' to the appropriate CSS will override this behavior and let divs appear next to each other.
